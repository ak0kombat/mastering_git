Initializing a git repository
git init //Initializing a local repository
git clone URL //Copying a repository from git hub using the repo url

To Stage file
git add . //stage everything in a folder but incase of a subfolder adds only the contents inside the subfolder leaving the files in the main directory
git add --all //stages everything even while working inside a subfolder
git add -A // same as --all
git add filename //a to stage a specific file
git add * //stages everything except deleted files
git add *.txt //Stages everything thats a txt file(staging file specific)

To unstage a file
git reset

Identity Confirmation
git config --global user.email "ak0kombat@gmail.com"
git config --global user.name "Akash"
git config --global --replace user.name "Akash"(To replace the name if configured already)

To commit the changes
git commit -m "Message to describe the version"

To check the status of the repo
git status

To check the changes done on the repo
git log
git log --oneline

Rollback after a commit
git reset HEAD~

To remove a file and stage the deletion
git rm filename
git rm -r <Folder> //to remove a Folder
To rollback a change including deleted file
git reset --hard


BRANCHING

git branch //Shows the list of branches in the repo
git branch <branchname> //To create a new branch
git checkout <branchname> //To access a branch
git checkout <commit ID> //To rollback to a previous commit
git branch --delete <branch name>

git diff <commit ID 1> <commit ID 2> //Displays the difference btw the two commits

CONNECTING TO THE REMOTE REPO
git remote add origin <REPO URL>
git push origin main <Branch name>

git push origin --delete <branch name>
git push origin 

Fetching and merging
git fetch //fetches the changes made in the remote repository without editing it in the local repo
git merge //merges the changes done in remote to local repo

Pulling
git pull origin branch_name //performs merge and fetch all together

RESTORE
git restore <file name> //restores the file to the last commit state
git restore . //restores all the file
git restore --staged <file name> //removes the file from the stagging area but keeps it in the working directory
git restore --staged . //removes all the files from the staging area

STASHING
git stash //moves the working progress to backrooms when we need to switch another branch
git stash pop //brings the progress from backrooms to pwd
git stash clear // removes the progress from the backrooms
git stash apply //brings the progress but doesn't clear the hash values

REVERT
git revert <desired commit ID> //Safely reverting the change
